Use standard interfaces for joints because they are relevant for dynamic behaviour.

By splitting the standard interfaces from other type, the users are able to inherit this class and simply create small "simulation" with desired dynamic behaviour. The advantage over using Gazebo is that enables "quick & dirty" tests of robot's URDF and controllers.
